====================================
2. סטטוס טכני – שכבת אפליקציה קיימת
====================================
2.1 טכנולוגיות
----------------
- HTML + CSS + JavaScript (ES Modules, ללא Framework).
- Three.js עבור 3D (חללית חיה + סטודיו 3D).
- Vite עבור build ו-dev server.
- localStorage לאחסון:
  - צי חלליות.
  - הגדרות משתמש (theme, נגישות, ביצועים).
  - Session למשתמש מחובר.
  - Cooldown לטופס הבקשה.

2.2 מבנה כללי
--------------
- index.html – שלד דף אחד עם כל המקטעים והמודאלים.
- style.css – כל ה-Design System, רכיבים, סטיילינג ורספונסיביות.
- script.js – כל הלוגיקה:
  - Session + Login + הרשאות.
  - טפסים (מגדל הרכבה + בקשת משתמש).
  - ניהול צי.
  - תצוגות 3D (חיבור ל-3d/*.js).
- תיקיית 3d/ – כל קוד המודל התלת־ממדי (factory/editor/materials/utils/scene).
- vite.config.js + package.json – הגדרות build ותלויות.

2.3 תצוגת 3D
-------------
- מודל חללית יחיד, עם פרמטרים בסיסיים כמו:
  - shipShape (classic/sleek/heavy/ring).
  - bodyLength, bodyRadius, wingSpan, wingAngle, engineSize, engineGlow, cockpitTint, decals, alienTechLevel.
- תצוגה חיה ב-builder + תצוגה מורחבת ב-studio3d:
  - זום, סיבוב, presets לזוויות מצלמה, צירים, מסך מלא, צילום.

6. התאמה לאפיון יעד – פערים עיקריים
====================================
מה באפיון נוסף ועדיין לא נמצא בקוד (מצב פתוח):
1) עורך חללית מודולרי (Ship Parts)
   - מעבר מצורה אחת + סליידרים כלליים ללוגיקה חלקית:
     - Nose, Body, Wings, Engines, Cockpit, Tail, Weapons, Cargo, Sensors, Landing Gear, Shields, Alien Add-ons.
   - UI מבוסס "ערוך חרטום / גוף / כנפיים / מנועים / קוקפיט":
     - כפתור לכל חלק + פתיחת פרמטרים רלוונטיים בלבד.
   - הדגשת חלק נבחר ב-3D (Highlight Part / Outline).
   - תמיכה ב-Toggle/Symmetry לכל חלק (בעיקר כנפיים/מנועים).

2) מבנה config3D חדש ומודולרי
   - מעבר ל:
     - config3D.parts.nose/body/wings[]/engines[]/cockpit/tail
     - config3D.style (צבעים/חומרים)
     - config3D.fx (glow, alienTechLevel, אפקטים נוספים)
   - מיגרציה מהפורמט הקיים:
     - בעת טעינת fleet:
       - אם אין parts → בניה אוטומטית ממבנה 0.6.0 (bodyLength, wingSpan וכו').

3) אחסון מורחב
   - הוספת fleet-v2 + backups.
   - users-v1 לניהול משתמשים נוספים (מעבר ל-hard-coded NM/MP).

4) Admin UI
   - לשונית/מסך ייעודי ל-MP:
     - ניהול משתמשים.
     - Debug tools.
     - סטטיסטיקות בסיסיות (כמות חלליות, שימוש, גרסאות וכו').
