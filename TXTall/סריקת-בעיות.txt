דוח סריקת בעיות – MP-Nadav-Web-2000
====================================

חלק 1 – מטרת המסמך
--------------------
1. מסמך זה מרכז את בעיות המערכת הידועות כרגע ואת מצב הטיפול בהן.
2. הדוח נכתב עבור הפרויקט SpaceYard (`MP-Nadav-Web-2000`).
3. הפוקוס כאן הוא על בעיות Build, Deployment (Vercel), קונפיגורציית Node, ושגיאות קוד.

חלק 2 – מצב קובצי הפרויקט הרלוונטיים
------------------------------------
4. `package.json` – מוגדר עם:
5.   name: "spaceyard"
6.   scripts: "dev": "vite", "build": "vite build", "preview": "vite preview"
7.   dependencies: "three": "^0.161.0"
8.   devDependencies: "vite": "^5.4.0"
9.   engines: { "node": "24.x" } – כדי להתאים לדרישת Vercel.
10. `vite.config.js` – מגדיר:
11.   root: "."
12.   build.outDir: "dist"
13.   build.assetsDir: "assets"
14.   build.emptyOutDir: true
15. `script.js` – קובץ ה־JS הראשי של האפליקציה, כולל:
16.   לוגיקת 3D, צי חלליות, הגדרות, Login, מודעות, טפסים ועוד.

חלק 3 – בעיית ה‑Build ב‑Vercel (שגיאת תחביר ב‑script.js)
---------------------------------------------------------
17. השגיאה הראשונית שחזרה מ‑Vercel הייתה:
18.   [vite:build-import-analysis] script.js: Failed to parse source for import analysis because the content contains invalid JS syntax.
19. ההסבר של Vercel/rollup:
20.   אם יש JSX, צריך סיומת .jsx או .tsx.
21. בפועל, לא השתמשת ב‑React/JSX, אלא במחרוזות HTML רגילות.
22. המקור לשגיאה היה template string בתוך template string:
23.   `${mission ? `<br />תיאור המשימה: ${mission}` : ""}`
24. זהו תחביר לא חוקי ב‑JavaScript רגיל (backtick בתוך backtick).
25. השורה שונתה לכתיבה תקינה:
26.   `${mission ? "<br />תיאור המשימה: " + mission : ""}`
27. אחרי תיקון זה, `script.js` הוא קוד JS חוקי לחלוטין מבחינת תחביר.

חלק 4 – בעיית גרסת Node ב‑Vercel
---------------------------------
28. Vercel התריע על:
29.   "Found invalid or discontinued Node.js Version: "18.x"."
30. הדרישה החדשה של Vercel:
31.   להשתמש ב‑Node 24: `"engines": { "node": "24.x" }`.
32. בקובץ `package.json` עודכן שדה engines בהתאם:
33.   לפני כן: "node": ">=18.0.0" → יצר אזהרה על שדרוג אוטומטי לגרסאות עתידיות.
34.   לאחר מכן: "node": "18.x" → Vercel סימן כגרסה מיושנת.
35.   עכשיו: "node": "24.x" → מיושר להנחיית Vercel העדכנית.
36. אזהרת גרסת Node אמורה להיפתר לאחר ש‑GitHub מעודכן וש‑Vercel בונה מחדש את הקומיט העדכני.

חלק 5 – בעיית Node / npm במחשב המקומי (Windows)
-----------------------------------------------
37. במחשב המקומי, במצב ההתחלתי:
38.   פקודות `node -v` ו־`npm -v` לא הוכרו (לא זוהו).
39. המשמעות:
40.   לא ניתן להריץ `npm install` או `npm run build` מקומית.
41.   לא ניתן לבדוק את שגיאות ה‑build בצורה עצמאית לפני שליחה ל‑Vercel.
42. זה גרם למצב שבו כל Deploy כושל, בלי יכולת לאמת build קודם אצלך.
43. הפתרון הנדרש:
44.   להתקין Node.js מהאתר הרשמי `https://nodejs.org` (גרסת LTS מומלצת).
45.   במהלך ההתקנה לסמן "Add to PATH".
46.   לאחר ההתקנה לפתוח CMD חדש, לבדוק:
47.     `node -v` ו־`npm -v` → אמורים להחזיר מספרי גרסה.
48. לאחר מכן, בתיקיית הפרויקט:
49.   `npm install`
50.   `npm run build`
51. אם build מקומי עובר, זה סימן טוב ש‑Vercel גם צריך לעבור (בהנחה שגרסת Node שם מתאימה).

חלק 6 – סטטוס script.js אחרי תיקונים
------------------------------------
52. סריקת הקובץ `script.js` על כל המופעים של התו `<` הראתה:
53.   כל ה‑HTML נמצא בתוך template string אחד המשמש ל‑innerHTML.
54.   לא נמצאו `import React`, `export default`, `const App =`, או JSX אחר.
55. אין `<div>`, `<section>`, `<button>` וכו' מחוץ למחרוזות רגילות.
56. כלומר: הקובץ אינו מכיל JSX, אלא רק JS רגיל ומחרוזות HTML.
57. לאחר שינוי שורת ה‑mission, אין עוד backticks מקוננים בתוך backticks.
58. בדיקת לינט (`ReadLints`) לא מצאה שגיאות בתחביר או בעיות ידועות אחרות.

חלק 8 – בעיות UX / כפילויות קוד

68. יש מספר האזנות למקש Escape (`document.addEventListener("keydown", ...)`) עבור:
69.   חלון ההגדרות (Settings Modal).
70.   חלון בקשת משתמש (Request Modal).
71.   חלון גלריה (Gallery Lightbox).
72.   חלון ברוך הבא (Welcome Modal).
73. זו אינה שגיאה, אך זוהי כפילות לוגית שניתן לאחד בעתיד.
74. ניהול המודאלים (פתיחה/סגירה) מבוסס על מספר מחלקות ו‑listeners דומים.
75. אפשר בעתיד ליצור "Modal Manager" מרכזי שיטפל בכל המודאלים בצורה אחידה.
76. הקובץ `script.js` ארוך מאוד (מעל 1300 שורות), מה שמקשה על תחזוקה:
77.   מומלץ לפצל בעתיד למודולים קטנים: auth.js, fleet.js, settings.js, ui.js וכו'.

חלק 9 – בעיות שאינן באג אבל חשוב לשים לב אליהן
----------------------------------------------
78. ה־Node Version ב‑Vercel סומנה כעת כ‑"24.x", אך ייתכן שעדיין:
79.   Vercel מריץ build על קומיט ישן שלא כולל את העדכון ב‑package.json.
80. חשוב לוודא ש:
81.   בוצע `git add package.json`, `git commit`, `git push origin main`.
82.   ב‑Vercel ה‑Deployment האחרון מצביע על הקומיט האחרון בסניף main.
83. אם הלוג האחרון עדיין מצביע על "18.x", כנראה מדובר ב‑Deployment ישן.
84. שימוש ב‑Node 24.x אפשרי ולא אמור לשבור את Vite 5.
85. עם זאת, מקומית אפשר לעבוד גם עם גרסת LTS (20/22), אין חובה ל‑24.

חלק 10 – סיכום מצב הבעיות כרגע
-------------------------------
86. בעיית הסינטקס ב‑`script.js` – **תוקנה**.
87. בעיית גרסת Node ש‑Vercel דרש (24.x) – **טופלה** ע"י עדכון `package.json`.
88. בעיית Node/npm לא מותקנים מקומית – דורשת התקנת Node על פי ההנחיות.
89. בעיות אבטחה (סיסמאות גלוייות בצד לקוח) – מוכרות, נסבלות לסביבת בילד.
90. בעיות כפילויות קוד וניהול מודאלים – אינן באג אבל נקודת שיפור עתידית.
91. השלב הבא החשוב מבחינתך:
92.   לוודא שהתקנת Node מקומית, שה‑build עובר אצלך (`npm run build`),
93.   ואז לבצע Push ל‑GitHub ול‑Redeploy ב‑Vercel.
94. כל שגיאה חדשה שתופיע בלוג ה‑build אחרי התיקונים הנ"ל תצריך דוח/עדכון נוסף.
