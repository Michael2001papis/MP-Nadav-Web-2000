## דוח מצב אתר SpaceYard

**תאריך יצירה**: 02/03/2026  
**סביבת פיתוח**: פרויקט Frontend סטטי מבוסס Vite + Three.js

---

### 1. טכנולוגיות וסטאק

- **Bundler/Dev Server**: `Vite` (גרסה ^5.4.0) מוגדר בקובץ `package.json` עם סקריפטים:
  - `npm run dev` – הרצת שרת פיתוח.
  - `npm run build` – בניית גרסת הפקה לתיקיית `dist`.
  - `npm run preview` – תצוגה מקומית של הבילד.
- **ספריות עיקריות**:
  - `three` (^0.161.0) – מנוע 3D להצגת ועריכת חלליות.
- **טכנולוגיות צד־לקוח**:
  - HTML אחד ראשי: `index.html` (שפה עברית, כיוון `rtl`).
  - CSS יחיד: `style.css` – עיצוב מלא לאתר, כולל התאמות מובייל/נגישות.
  - JS מודולרי: `script.js` + מודולי 3D בתקיית `3d/`.

---

### 2. מבנה כללי של האתר

- **עמודים/סקטורים עיקריים ב־`index.html`**:
  - `#home` – אזור Hero עם כפתור קריאה לפעולה "בוא נבנה חללית".
  - `#builder` – "מגדל ההרכבה" הכולל טופס מפורט לבניית חלליות (שם, משימה, צוות, מהירות, גודל, צבעים ועוד).
  - `#studio3d` – "סטודיו 3D" לתצוגה ויזואלית מלאה ותפעול מצלמה (זום, סיבוב, צירים, מסך מלא, צילום).
  - `#fleet` – רשימת "הצי שבנית" עם סיכומי חלליות, מיון, סינון וחיפוש.
  - `#gallery` – "גלריית השראה ושרטוטים" עם תמונות חיצוניות ושרטוטים וקטוריים.
- **תפריט ראשי**:
  - ניווט עליון (`header`) כולל לוגו, קישורי עוגן לסקטורים, וכפתור הגדרות (`⚙️ הגדרות`).
  - תפריט "המבורגר" למובייל (`nav-drawer`) עם אותם קישורים + גישה להגדרות.
- **פוטר**:
  - טקסט זכויות יוצרים דינמי המשתמש באלמנט `#year` שמעודכן לפי השנה הנוכחית.

### 3. פיצ'רים עיקריים בצד הלקוח (`script.js`)

- **ניהול צי חלליות**:
  - שמירה/טעינה של הצי מ־`localStorage` תחת מפתח `spaceyard-fleet-v1`.
  - יצירה, עדכון, מחיקה ושכפול חלליות.
  - ולידציה לטופס (שם חללית, מספר צוות, שם מפקד/ת).
  - הצגת סיכום מילולי עשיר לכל חללית (סוג משימה, יכולות, מערכות, חייזר מלווה, רמת סיכון, תיאור משימה).
  - הצגת כרטיסים בקטגוריית "הצי שלי" עם:
    - מיון לפי תאריך יצירה/מהירות/גודל/צוות/רמת סיכון.
    - סינון לפי סוג משימה.
    - חיפוש לפי שם חללית.
    - מחיקת חללית בודדת עם אישור.
    - סימון חללית בעריכה (סטייט Edit Mode).
- **ניהול הגדרות (Settings)**:
  - טאב "תצוגה": מצב צבעים (normal/alien/simple), הגדלת אותיות, הפחתת תנועה, ניגודיות גבוהה.
  - טאב "3D": סיבוב אוטומטי, מהירות סיבוב, איכות 3D, הצגת צירים.
  - טאב "צי ואחסון": שמירה אוטומטית, מגבלת כמות חלליות בצי, אישור לפני איפוס.
  - טאב "פרטיות": כפתור למחיקת כל הנתונים המקומיים (צי + הגדרות).
  - שמירה של ההגדרות ב־`localStorage` תחת `spaceyard-settings-v2` (עם תמיכה לאחור ל־V1).
  - יישום הגדרות על ה־DOM:
    - שינוי `data-theme` על ה־`body` (כולל `simple`/`alien`).
    - מחלקות `font-size-large`, `reduced-motion`, `high-contrast`.
    - עדכון הגדרות 3D דרך `threeContext.set3DOptions`.
- **תצוגה חיה של חללית**:
  - בעת שינוי הטופס, נוצר אובייקט חללית נורמלי ומעודכן טקסט הסיכום והתצוגה החיה.
  - שימוש ב־`createShip` ו־`buildConfig3DFromRaw` כדי לתרגם ערכי הטופס לקונפיגורציית 3D.
  - עדכון צבע דגש ב־CSS (`--accent`) לפי צבע החללית.
- **גלריית השראה**:
  - כרטיסי גלריה לחיצים/נגישים עם keyboard.
  - Lightbox מותאם לעברית:
    - הגדלת תמונה/שרטוט.
    - הצגת כותרת וטקסט מתוך הכרטיס.
    - סגירה בלחיצה על Overlay, כפתור `×`, או מקש Escape.
- **ניהול טוסטים**:
  - הודעות סטטוס (הצלחה/שגיאה/מידע) בתחתית המסך, כולל אנימציית Fade-out.

### 4. מנוע 3D ותצוגת חלליות (`3d/`)

- **`scene.js`**:
  - אתחול סצינת Three.js עם:
    - מצלמת Perspective, תאורה סביבתית + כיוונית + Rim Light.
    - `OrbitControls` עם Damping, מגבלת זום ופאן אופציונלי.
  - תמיכה ב:
    - `setShipGroup` – החלפת אובייקט החללית בסצנה.
    - `set3DOptions` – שליטה על סיבוב אוטומטי, מהירות, איכות רינדור, הצגת צירים, פאן.
    - `setCameraPreset` – Presets לזוויות מצלמה (חזית, צד, מלמעלה, איזומטרי).
    - `resetView`, `zoomIn`, `zoomOut`.
  - לולאת אנימציה רציפה עם רוטציה עדינה והנעה קלה בציר Y.
- **`shipFactory.js`**:
  - בנאי חלליות לפי סוג:
    - `classic`, `sleek`, `heavy`, `ring`.
  - בונה חלקים בסיסיים (גוף, קוקפיט, כנפיים, מנוע) ומשתמש בחומרי גלם מתוך `materials.js`.
  - מתאים מידות (אורך, רדיוס, כנפיים, מנוע) לפי הקונפיגורציה המספרית.
- **`shipEditor.js`**:
  - עדכון חללית קיימת לפי שינויי קונפיגורציה (סקייל, זוויות כנפיים, גודל מנוע, טינט קוקפיט).
- **`utils.js`**:
  - קונפיגורציית ברירת מחדל ל־3D (`defaultShipConfig3D`).
  - פונקציות עזר: `clamp`, `degToRad`, `mergeConfig3D`, ו־`buildConfig3DFromRaw` שמתרגם את ערכי הטופס למבנה קונפיגורציה נורמלי.

### 5. עיצוב, רספונסיביות ונגישות (`style.css`)

- **שפה וכיוון**:
  - `html` מוגדר עם `lang="he"` ו־`dir="rtl"`.
  - העיצוב והפריסה מותאמים ל־RTL (תפריט צד, Drawer, טקסטים).
- **מצבי עיצוב (Themes)**:
  - Normal (ברירת מחדל: רקע כהה חללי).
  - `data-theme="simple"` – מצב בהיר פשוט.
  - `data-theme="alien"` – מצב "חייזרי" עם צבעי ירוק/טורקיז חזקים.
- **נגישות/התאמות**:
  - מצבי `high-contrast` ו־`reduced-motion`.
  - כפתורים ורכיבים עיקריים בגובה מינימלי 44px.
  - פוקוס ברור ב־Keyboard (`:focus-visible`).
  - שימוש ב־`aria-label`, `aria-live`, `role=dialog`, `role=toolbar`, `aria-hidden` ועוד.
- **רספונסיביות**:
  - Media Queries ל־400px, 640px, 680px, 768px:
    - מעבר ניווט לתפריט המבורגר.
    - התאמת גריד של בונה החלליות, גלריה, וסטודיו 3D לגודל מסך.
    - התאמות לרוחב טפסים וכפתורים במובייל (פריסה בעמודה).

### 6. אחסון נתונים ומצבי ברירת מחדל

- **צי חלליות**:
  - קריאה ושמירה דרך `localStorage`:
    - מפתח צי: `spaceyard-fleet-v1`.
  - בעת טעינה ראשונית ללא צי, נוצרת חללית דמו (`אוריון פרו`) כדי להמחיש שימוש.
- **הגדרות משתמש**:
  - מפתח הגדרות חדש: `spaceyard-settings-v2`.
  - מפתח ישן נתמך: `spaceyard-settings-v1` (מומר ל־V2 אם קיים).
  - שמירת הגדרות נעשית רק בצד לקוח, אין תקשורת שרת.
- **כפתור "מחק את כל הנתונים המקומיים"**:
  - מנקה את שני מפתחות ההגדרות ואת מפתח הצי, ומבצע ריענון מלא של הדף.

---

### 7. מצב ריצה ובנייה

- **כדי להריץ מקומית** (לפי `package.json`, ללא שינוי בקוד):
  - התקנת תלויות (פעם ראשונה):  
    - `npm install`
  - הרצת פיתוח:  
    - `npm run dev` (Vite dev server, ברירת מחדל לרוב על `http://localhost:5173`).
- **כדי לבנות גרסת הפקה**:
  - `npm run build` – יוצר תיקיית `dist` עם קבצים סטטיים.
  - `npm run preview` – תצוגת ה־build המקומית.
- **Side effects ידועים בזמן ריצה**:
  - שימוש ב־`localStorage` בלבד; אין קריאות לרשת/API חיצוני מלבד טעינת תמונות רקע מה־CDN (Unsplash) בגלריה.

---

### 8. הערכת מצב כללית

- **כיסוי פיצ׳רים**:  
  האתר ממומש עם חוויית משתמש עשירה: בונה חלליות אינטראקטיבי, תצוגת 3D עם שליטה מתקדמת, צי חלליות עם ניהול מלא, ומסך גלריית השראה.

- **איכות קוד**:  
  הקוד מודולרי יחסית (הפרדת לוגיקת 3D ל־תקיית `3d/`), מנצל היטב את Three.js ו־Vite, ומכיל טיפול סביר בשגיאות/חוסר זמינות של 3D (Fallbacks ו־Toasts).

- **נגישות ותמיכה בעברית**:  
  הטקסטים, ה־`dir="rtl"` וה־ARIA נראים מסודרים, עם דגש על קריאות בעברית ושימוש ב־Keyboard.

- **סיכום**:  
  האתר נמצא במצב פיתוח בשל ומוכן להרצה/דמו, עם תשתית בנייה ו־3D יציבה, ללא תלות בשרת Backend.
