מטרת המסמך היא להגדיר:

═══════════════════════════════════════════════════════════════════════════════
  חלק 2 – הבנת מבנה הפרויקט
═══════════════════════════════════════════════════════════════════════════════

הפרויקט מורכב משלושה אזורים מרכזיים:

--------------------------------------------------------------------------------
  1. שכבת ממשק (Frontend UI)
--------------------------------------------------------------------------------

  קבצים מרכזיים:
    • index.html   – מבנה האתר, כל הסקשנים
    • style.css    – כל העיצוב
    • script.js    – כל הלוגיקה הראשית

  תפקידם:
    • מבנה האתר
    • כל הסקשנים (Hero, Builder, Studio 3D, Editor 3D, Fleet, Gallery, פרופיל, וכו')
    • כל הלוגיקה הראשית (הגדרות, Auth, טופס חללית, צי, גלריה, טוסטים)
    • כל העיצוב (משתנים, רספונסיבי, RTL, מצבי צבע)

  חשוב להבין:
    • האתר הוא Single Page Application בסיסי.
    • כל הסקשנים נמצאים באותו HTML.
    • אין פריימוורק – JavaScript ונילה (ES Modules).

--------------------------------------------------------------------------------
  2. מנוע התלת־ממד (Three.js)
--------------------------------------------------------------------------------

  תיקייה:  3d/

  תפקידה:  ניהול כל מערכת התלת־ממד באתר.

  קבצים עיקריים:

    קובץ          | תַפקיד
    --------------|----------------------------------------------------------
    scene.js      | יצירת סצנה, מצלמה, רינדור, OrbitControls, אנימציה
    shipFactory.js| יצירת מודלים של חלליות (גאומטריות, צורות שונות)
    shipEditor.js | שינוי חללית קיימת (צבעים, גדלים, זוהר מנוע)
    materials.js  | חומרים (metal, matte, glass, alien)
    utils.js      | פונקציות עזר (קלמפ, רדיאנים, mergeConfig3D)
    editor3d.js   | מערכת עורך החלליות (חלקים, כלים, blueprint, שמירה/טעינה)

--------------------------------------------------------------------------------
  3. נכסים (Assets)
--------------------------------------------------------------------------------

  תיקייה:  public/assets/

  תכולה:
    • images/     – לוגו האתר, תמונת רקע
    • models/     – מודלים עתידיים (.gitkeep)
    • hdr/        – HDR עתידי (.gitkeep)

  השרת של Vite מגיש אותם ישירות (אין צורך ב־import לנתיבים ב־public/).


═══════════════════════════════════════════════════════════════════════════════
  חלק 3 – חוקי עבודה בפרויקט
═══════════════════════════════════════════════════════════════════════════════

בעת עבודה על האתר יש לשמור על הכללים הבאים.

--------------------------------------------------------------------------------
  כלל 1 – לא לשבור את ה־Entry
--------------------------------------------------------------------------------

  הקובץ המרכזי של המערכת הוא:  script.js

  זהו הקובץ שמאתחל:
    • UI (ניווט, מודלים, טוסטים, גלריה, lightbox)
    • 3D (תצוגה בבונה, סטודיו, עורך)
    • LocalStorage (הגדרות, צי, פרופיל, טקסטים)
    • Auth (דמו – אורח/עסקי/מפתח)
    • Fleet (שמירה, טעינה, סינון, מיון)
    • טפסים (מגדל הרכבה, פרופיל עסקי, טקסטים, פנייה למפתח)

  אסור:
    • למחוק פונקציות מרכזיות (loadSettings, saveSettings, loadFleet, renderFleetList, וכו')
    • לשנות שמות מפתחות אחסון (ראו כלל 2)
    • לשנות מבנה נתונים (אובייקט חללית, אובייקט הגדרות) בלי בדיקה מלאה ומיגרציה.

--------------------------------------------------------------------------------
  כלל 2 – לא לשנות LocalStorage Keys
--------------------------------------------------------------------------------

  המפתחות הבאים הם חלק ממערכת הנתונים:

    spaceyard-fleet-v1
    spaceyard-settings-v2
    spaceyard-settings-v1
    spaceyard-auth-user-v1
    spaceyard-welcome-shown-v1
    spaceyard-business-profile-v1
    spaceyard-dev-text-v1
    spaceyard-editor-designs-v1

  שינוי שלהם עלול:
    • למחוק נתוני משתמשים
    • לשבור טעינת נתונים
    • לשבור מיגרציות (למשל מ־settings v1 ל־v2)

  הוספת מפתח חדש: אפשרי רק עם גרסה חדשה (למשל spaceyard-xyz-v2) ותיעוד.

--------------------------------------------------------------------------------
  כלל 3 – לא לשבור את מערכת ה־3D
--------------------------------------------------------------------------------

  מערכת התלת־ממד רגישה.

  יש להקפיד על:
    • שימוש נכון ב־Three.js (Scene, Camera, Renderer, dispose כשנפטרים מאובייקטים)
    • שמירה על מבנה הפונקציות ש־script.js קורא להן (initScene, createShip, updateShipFromConfig, initEditor3D)
    • לא ליצור עומס GPU מיותר (גאומטריות, אורות, חלקיקים)

  בעת שינוי:
    • לבדוק FPS (במכשיר חלש ובמובייל)
    • לבדוק מובייל (תצוגה, זום, סיבוב)
    • לכבד qualityMode (auto / high / low) ב־scene.js


═══════════════════════════════════════════════════════════════════════════════
  חלק 4 – שיפורים מומלצים למערכת
═══════════════════════════════════════════════════════════════════════════════

תבסס על הסריקה.

--------------------------------------------------------------------------------
  שיפור 1 – פירוק script.js
--------------------------------------------------------------------------------

  הקובץ כרגע:  ≈ 1700+ שורות. זה לא בר־תחזוקה.

  מומלץ לפרק למודולים בתיקייה  modules/  לדוגמה:

    modules/
      auth.js
      fleet.js
      builder.js
      gallery.js
      settings.js
      ui.js
      business.js
      storage.js

  הקובץ הראשי יהפוך ל:  script.js → bootstrap בלבד (ייבוא ואתחול).

--------------------------------------------------------------------------------
  שיפור 2 – ריכוז קבועים
--------------------------------------------------------------------------------

  קבועים מפוזרים בקוד (ב־script.js ו־3d/).

  מומלץ ליצור קובץ  config.js  או  constants.js  שיכלול:
    • מפתחות localStorage
    • הגדרות ברירת מחדל (DEFAULT_SETTINGS)
    • הגדרות 3D (איכות, מצלמה)
    • הגדרות UI (תוויות, טקסטים)

--------------------------------------------------------------------------------
  שיפור 3 – מערכת DOM מרכזית
--------------------------------------------------------------------------------

  יש עשרות קריאות  getElementById  ו־querySelector  בראש script.js.

  מומלץ ליצור מבנה מרכזי (DOM Map) שירכז את כל האלמנטים במקום אחד, לדוגמה:

    const dom = {
      year: document.getElementById("year"),
      shipForm: document.getElementById("ship-form"),
      fleetList: document.getElementById("fleet-list"),
      ...
    };

  זה מקל על תחזוקה, Debug ושינוי מבנה HTML.


═══════════════════════════════════════════════════════════════════════════════
  חלק 5 – שיפור ביצועים
═══════════════════════════════════════════════════════════════════════════════

--------------------------------------------------------------------------------
  תמונת הרקע
--------------------------------------------------------------------------------

  התמונה הנוכחית (public/assets/images/...) עלולה להיות כבדה.

  מומלץ:
    • המרה ל־WebP
    • דחיסה (רוחב מקסימלי ~1920px)
    • גרסה נפרדת למובייל (קטנה יותר)
    • Lazy loading או background עם size/priority מתאימים

--------------------------------------------------------------------------------
  מנוע 3D
--------------------------------------------------------------------------------

  במכשירים חלשים:
    • להוריד איכות (qualityMode: low כבר קיים)
    • להקטין pixel ratio (כבר ב־scene.js – setPixelRatioFromQuality)
    • להפחית פוליגונים או LOD כשהמצלמה רחוקה

--------------------------------------------------------------------------------
  גלריה
--------------------------------------------------------------------------------

  כבר יש:  loading="lazy"  על תמונות.

  ניתן להוסיף:
    • Skeleton loading
    • Placeholder blur (blur-up)
    • גודל קבוע ל־img כדי למנוע layout shift


═══════════════════════════════════════════════════════════════════════════════
  חלק 6 – אבטחת מידע
═══════════════════════════════════════════════════════════════════════════════

  האימות הנוכחי (AUTH_USERS) נמצא ב־Frontend. זה לא אבטחה אמיתית – מתאים לדמו בלבד.

  אם האתר יעבור לפרודקשן עם משתמשים אמיתיים, יש להוסיף:
    • שרת Backend (למשל Node.js + Express)
    • Database לאחסון משתמשים ונתונים
    • מערכת אמיתית תכלול:
      – Hash לסיסמאות (למשל bcrypt)
      – Sessions או JWT
      – API מאובטח (HTTPS, CORS, rate limiting)


═══════════════════════════════════════════════════════════════════════════════
  חלק 7 – נגישות
═══════════════════════════════════════════════════════════════════════════════

  האתר כבר תומך בחלק מהתקנים (aria-label, aria-controls, aria-expanded, role).

  יש להמשיך לשמור על:
    • aria-label על כפתורים וקלטים
    • aria-controls / aria-expanded במודלים ובטאבים
    • role="dialog", role="tab", role="tabpanel" בהגדרות

  בנוסף יש לוודא:
    • ניווט מקלדת (Tab, Enter, Escape לסגירת מודלים)
    • קריאות טקסט (ניגודיות, גודל גופן)
    • מצב high contrast (body.high-contrast כבר קיים ב־CSS)


═══════════════════════════════════════════════════════════════════════════════
  חלק 8 – פריסה (Deployment)
═══════════════════════════════════════════════════════════════════════════════

  האתר הוא Static Build. כלומר: אין צורך בשרת Node בפרודקשן.

  שלבי פריסה:
    1.  npm run build   → יצירת  dist/
    2.  העלאה של תוכן  dist/  לשרת סטטי

  ניתן לפרוס ב:
    • Vercel
    • Netlify
    • GitHub Pages
    • Cloudflare Pages

  אם האתר יושב בתת־תיקייה (למשל example.com/spaceyard/), להגדיר ב־vite.config.js:
    base: "/spaceyard/"

  חלק 10 – בדיקות לפני העלאה
═══════════════════════════════════════════════════════════════════════════════

  לפני העלאה לשרת יש לבדוק:

  □ האתר נטען ללא שגיאות קונסול
  □ כל הסקשנים עובדים (גלילה, קישורי ניווט)
  □ מנוע 3D עובד:
      □ מצלמה, זום, סיבוב (בונה + סטודיו)
      □ יצירת חללית מהטופס
      □ עורך 3D (למשתמש עסקי/מפתח)
  □ localStorage עובד:
      □ שמירת צי
      □ הגדרות (תצוגה, 3D, פרטיות)
      □ משתמש (כניסה/יציאה, פרופיל עסקי)
  □ גלריה נטענת:
      □ תמונות
      □ Lightbox
  □ טפסים: שמירת פרופיל, טקסטים (הגדרות → טקסטים עיסקי), פנייה למפתח (mailto)
  □ רספונסיביות בסיסית (רוחב מסך קטן)

  חלק 11 – שיפורים עתידיים אפשריים
═══════════════════════════════════════════════════════════════════════════════

  האתר יכול להתפתח למערכת גדולה יותר. אפשרויות:

    • מערכת משתמשים אמיתית     – Backend + DB, הרשאות
    • שמירת צי בענן             – User Fleet Storage, סנכרון בין מכשירים
    • שיתוף חלליות              – Community Gallery, ייצוא/ייבוא
    • מודלים 3D אמיתיים         – GLTF/GLB במקום גאומטריות פרימיטיביות
    • מנוע פיזיקה               – למשל cannon.js, ammo.js (התנגשויות, כבידה)

  חלק 12 – מסקנה מקצועית
═══════════════════════════════════════════════════════════════════════════════

  הפרויקט SpaceYard הוא מערכת  Interactive 3D Web Application  עם:
    • UI מלא (Hero, Builder, Studio 3D, Editor 3D, Fleet, Gallery, פרופיל, הגדרות)
    • 3D Editor
    • Fleet Manager
    • Gallery והשראה

  המערכת יציבה, אך זקוקה:
    • פירוק קוד (מודולים)
    • אופטימיזציה (תמונות, 3D במובייל)
    • ארכיטקטורה מודולרית (קבועים, DOM Map)

  כדי להפוך למערכת מקצועית וברת־תחזוקה – יש לעקוב אחרי הוראות עבודה אלו ולבצע
  שינויים בצורה מבוקרת ובטוחה.
