## SpaceYard – בונה החלליות שלך 🚀

יישום דמו/תצוגה פרונט־אנד מלא, שמדמה "מעבדת חלליות" עם:
- בונה חלליות עם טופס עשיר
- תצוגת 3D חיה (Three.js)
- סטודיו 3D נפרד
- עורך חלליות 3D מתקדם
- ניהול צי חלליות, גלריית השראה, וכלי עזר

אין שרת/בקאנד – כל הנתונים נשמרים מקומית בדפדפן באמצעות `localStorage`.

---

## טכנולוגיות

- **Vite 5** – פיתוח ובנייה (dev/build/preview)
- **JavaScript (ES Modules)** – ללא React / Vue, קוד ונילה מודולרי
- **Three.js** – סצנות 3D, תצוגה חיה, סטודיו ועורך
- **CSS מודרני** – עיצוב רספונסיבי, RTL, מצבי צבע, אנימציות

---

## הרצה מקומית

```bash
npm install
npm run dev     # פיתוח – שרת Vite
npm run build   # בניית production לתיקיית dist
npm run preview # תצוגת הבילד לאחר build
```

ברירת מחדל: Vite מאזין על `http://localhost:5173` (או פורט פנוי אחר).

## מבנה הפרויקט וקבצים

### קבצי שורש

- **`index.html`**  
  - מסמך ה־HTML הראשי של האפליקציה.  
  - כולל:
    - `<link rel="icon" href="/assets/images/ship-logo.png">` – אייקון האתר.
    - טעינת הסגנונות: `style.css`.
    - טעינת הלוגיקה הראשית: `<script type="module" src="script.js"></script>`.
  - מבנה עיקרי:
    - `header.site-header` – לוגו, ניווט עליון, כפתורי הגדרות/כניסה/יציאה.
    - `section#home.hero` – אזור ה־Hero (כותרת ראשית, כפתור "בונה החלליות שלך", פאנל טלו־מטריה).
    - `section#builder` – מגדל ההרכבה (טופס בניית חללית + תצוגת 3D חיה).
    - `section#studio3d` – סטודיו 3D עם canvas נפרד וכלי מצלמה/זום/צילום.
    - `section#editor3d` – עורך חלליות 3D (מופיע רק למשתמש עסקי/מפתח).
    - `section#fleet` – רשימת הצי.
    - `section#gallery` – גלריית השראה (תמונות Unsplash + ציורי CSS).
    - `section#business-profile` – פרופיל משתמש עסקי + טקסטים מתכנתים.
    - `section#developer-text-panel` – פאנל טקסטים למפתח.
    - `section#user-requests` – טופס "ניהול משתמשים – פנייה למפתח" (mailto).
    - `section#tools` – רשימת כלי תכנון מומלצים (Fusion, Blender וכו').
    - `section#about` – מידע על העורך 3D ואיך להשתמש בו.
    - `footer.site-footer` – כיתוב זכויות, לוגו תחתון.

- **`style.css`**  
  קובץ ה־CSS המרכזי של כל המערכת:
  - הגדרות משתני צבע (`:root`, `--bg-*`, `--q-*`, `--text-*`).
  - הגדרת **רקע כללי**:
    - שכבת גרדיאנט (`--bg-gradient`) מעל
    - תמונת הרקע:  
      `"/assets/images/Website background image/Gemini_Generated_Image_eavh8eavh8eavh8e.png"`  
      כ־background fixed על `body`.
  - תמיכה במצבים:
    - `body[data-theme="simple"]` – מצב בהיר.
    - `body[data-theme="alien"]` – מצב צבעוני/חייזרי.
    - `body.reduced-motion` – הפחתת אנימציות.
    - `body.high-contrast` – מצב ניגודיות גבוהה.
  - עימוד מלא לכל הרכיבים:
    - כותרת עליונה (`.site-header`, nav, drawer למובייל).
    - מודל הגדרות (`.settings-modal`, טאב־בר, שדות).
    - אזור ה־Hero (`.hero`, `.hero-panel`, כפתורי CTA).
    - טופס מגדל ההרכבה (`.ship-form`, `.builder-accordion`, range וכו').
    - פאנל תצוגת חללית וסטטיסטיקות (`.ship-preview-panel` ועוד).
    - רשימת הצי (`.fleet-list`, `.fleet-card`).
    - סטודיו 3D (`.studio3d-*`).
    - עורך 3D (`.editor3d-*`).
    - גלריית השראה + lightbox (`.gallery-*`).
    - טוסטים (`.toast-*`).
    - פרופיל עסקי (`.profile-*`), פאנל מפתח, טפסי יצירת קשר.
    - footer, grid, cards, about וכו'.

- **`script.js`**  
  קובץ הלוגיקה הראשי (ונילה JS, מודול אחד גדול) – אחראי על:
  - **הגדרות ושמירה מקומית**:
    - קריאה/כתיבה של `localStorage` להגדרות תצוגה, הגדרות 3D, צי חלליות, טקסטים, פרופיל עסקי.
    - עדכון DOM לפי ההגדרות (theme, font-size, reduced motion, axis, quality mode וכו').
  - **"אימות" דמו וסטטוסים**:
    - שכבת `#auth-overlay` (ברוכים הבאים + טופס התחברות).
    - אובייקט `AUTH_USERS` עם משתמשי דמו (אורח/עסקי/מפתח) – מיועד לדמו בלבד, ללא אבטחה אמיתית.
    - שמירת role נוכחי (אורח/עסקי/מפתח) ב־`localStorage`, והצגת/הסתרת אזורים:
      - כפתורי כניסה/יציאה.
      - פאנל עסקי (`.business-only-section`).
      - פאנל מפתח (`.developer-only-section`).
      - ניווט לעורך 3D (`.editor-access-only`).
  - **ממשק משתמש כללי**:
    - תפריט מובייל (drawer).
    - פתיחה/סגירה של מודל הגדרות.
    - טוסטים להודעות למשתמש.
    - גלריה + lightbox להגדלת כרטיסים.
  - **בונה החלליות / מגדל ההרכבה**:
    - ולידציה של הטופס (שם חללית, מספר צוות וכו').
    - בניית אובייקט חללית לפי השדות (סוג, מהירות, גודל, יכולות, מערכות, "חייזר מלווה").
    - עדכון תקציר טכני + תצוגת סטטיסטיקות (bars, pills).
    - אינטגרציה עם צי החלליות + תצוגת 3D.
  - **ניהול צי חלליות (`#fleet`)**:
    - שמירה/טעינה של רשימת חלליות מ־`localStorage`.
    - סינון/מיון (מהירות/גודל/צוות/סיכון).
    - חיפוש לפי שם.
    - בחירה, מחיקה, איפוס צי.
  - **אינטגרציה עם Three.js**:
    - אתחול תצוגת 3D בבונה (`#ship-canvas`).
    - סטודיו 3D (`#studio-canvas`).
    - טעינה/סנכרון מידע בין הטופס לבין הסצנות (באמצעות מודולי `3d/`).
  - **פרופיל עסקי וטקסטים**:
    - עדכון כרטיס פרופיל, צבע זיהוי, משימה מועדפת.
    - טופס טקסטים למשתמש עסקי + טקסטים למפתח (שנשמרים מקומית).
    - יישום הטקסטים המעודכנים על אלמנטים ב־`index.html`.
  - **טופס "ניהול משתמשים – פנייה למפתח"**:
    - ולידציה בסיסית.
    - הרכבת קישור `mailto:` עם הנתונים של המשתמש לשליחת אימייל.

- **`vite.config.js`**  
  קובץ קונפיגורציה מינימלי ל־Vite:
  - `root: "."` – שורש הפרויקט.
  - `build.outDir = "dist"` – תיקיית build.
  - `build.assetsDir = "assets"` – תיקיית משאבים בתוך `dist`.
  - `emptyOutDir: true` – ניקוי תיקיית build לפני בנייה.

- **`package.json`**  
  הגדרות NPM:
  - `"name": "spaceyard"`, `"version": "0.6.0"`.
  - סקריפטים:
    - `"dev": "vite"`
    - `"build": "vite build"`
    - `"preview": "vite preview"`
  - תלויות:
    - `"three": "^0.161.0"`
    - `"vite": "^5.4.0"` (devDependency).

## מודולים בתיקיית `3d/`

כל הקבצים האלו עובדים יחד כדי לייצר את חלליות ה־3D, גם בתצוגה החיה וגם בסטודיו/עורך.

- **`3d/scene.js`**  
  - אתחול סצנת Three.js בסיסית:
    - יצירת `Scene`, `PerspectiveCamera`, `WebGLRenderer`.
    - הגדרת אור, רקע, Grid/Axis (לפי הגדרות).
    - הוספת OrbitControls ותמיכה בזום/סיבוב/הזזה.
  - פונקציות עיקריות (שנקראות מ־`script.js`):
    - `initScene(canvas, options)` – אתחול סצנה על `<canvas>` עם פרמטרים (איכות, צירים, auto-rotate, וכו').
    - עדכון גודל renderer על שינוי גודל חלון.
    - לולאת אנימציה (`requestAnimationFrame`) שמסתובבת את החללית / מפעילה תנועה קלה (float).

- **`3d/shipFactory.js`**  
  - אחראי ל"ייצור" חללית חדשה מ־geometries בסיסיים:
    - גוף (גליל/קופסה וכו').
    - כנפיים.
    - מנועים.
    - קוקפיט.
  - בונה מבנה מורכב מ־`Mesh`‑ים שונים, לפי קונפיגורציה:
    - צורת חללית (`shipShape3D`: classic / sleek / heavy / ring).
    - גודל גוף/כנפיים/מנועים (sliders בטופס).
    - סוג חומר/חומרי גלם (מתכתי, מט, זכוכית, Alien).
  - מחזיר אובייקט חללית שניתן להוסיף לסצנה.

- **`3d/shipEditor.js`**  
  - עדכון חללית קיימת במקום ליצור חדשה:
    - שינוי צבעים (`primaryColor3D`, `secondaryColor3D`, `bodyColor3D` וכו').
    - שינוי גדלים, זוויות, ואפקט זוהר מנוע (`engineGlow3D`).
  - משמש כדי לסנכרן בין טופס מגדל ההרכבה לבין האובייקטים ב־Scene.

- **`3d/materials.js`**  
  - מרכז יצירת החומרים (Materials) לחללית:
    - בניית `MeshStandardMaterial`/`MeshPhysicalMaterial`/אחרים בהתאם לסוג חומר שבחר המשתמש.
    - שימוש בצבעים שהגיעו מהטופס.
  - מאפשר להחליף חומר בקלות לפי מצב (מתכתי, זכוכית, Alien).

- **`3d/utils.js`**  
  - פונקציות עזר:
    - קלמפ לטווחים, המרה מעלות↔רדיאנים.
    - בניית אובייקט קונפיגורציה מלא מהטופס (כולל ערכי ברירת מחדל).
    - בדיקה/טיפול במצבים של reduced motion, איכות נמוכה במכשירים חלשים, ועוד.
  - מרוכז פה כדי לא לפזר לוגיקת עזר על פני כל המודולים.

- **`3d/editor3d.js`**  
  - עורך 3D המתקדם:
    - סצנת Three.js נפרדת לעורך.
    - ניהול "חלקים" של חללית (גוף, כנפיים, מנועים, קוקפיט) עם בחירה/הזזה/סיבוב/סקייל.
    - תצוגת blueprint (canvas‑ים ל־top / side / front).
    - שמירה/טעינה של עיצובים ל־`localStorage`:
      - כפתורי `שמור` / `טען`.
      - רשימת עיצובים זמינים ב־`<select>` בעורך.
    - שילוב עם הגדרות כמו רשת/תאורה, מצב "חללית / סצנה / שרטוט" (הטאבים העליונים).

## הערות אבטחה / אופן שימוש

- "אימות המשתמשים" בפרויקט זה הוא **דמו בלבד**:
  - שמות המשתמש והסיסמאות מוגדרים בצד לקוח בקובץ `script.js`.
  - אין שרת שמאמת או שומר נתונים – כל השליטה היא בצד הדפדפן.
  - לא מתאים למערכת ייצור אמיתית, אלא כהדגמה UX/UI ולוגיקה פרונט־אנד.
- כל הנתונים (צי, טקסטים, פרופיל, עיצובים ב־3D) נשמרים ב־`localStorage`:
  - אין שמירת מידע על השרת.
  - נוח לניסויים ולדמו – הנתונים קשורים רק לדפדפן ולמחשב שבו משתמשים.

## כיווני המשך ושיפור

- פירוק `script.js` למודולים קטנים יותר (settings, auth, fleet, ui, business/dev וכו').
- הוספת שרת אמיתי (Node/Express למשל) אם רוצים להרחיב לפרויקט ייצור:
  - ניהול משתמשים והרשאות בצד שרת.
  - שמירת צי חלליות בפרסיסטנטיות אמיתית (DB).
  - API לטעינת מודלי 3D אמיתיים (GLTF/GLB).
- אופטימיזציה של תמונת הרקע והגלריה עבור רשתות איטיות.
